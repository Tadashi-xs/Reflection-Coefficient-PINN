# Reflection Coefficient PINN

## Структура проекта

```text
.
├── generate_data.py                # генерация синтетической выборки
├── requirements.txt                # зависимости
├── storage/                        # сгенерированные датасеты
├── visualization/                  # сохраненные графики и метрики
└── PINN/
    ├── PhysicsInformedNN.ipynb
    └── saved_models/               # сохраненные модели
```

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Генерация синтетической выборки

Базовый запуск (например, для `num_samples=30000`):

```bash
python generate_data.py --num_samples 30000
```

Такой запуск создаст файл:

```text
storage/dataset_30k.csv
```

Количество объектов должно быть кратно `1000`, потому что имена датасетов строятся в формате

```text
dataset_{количество_тысяч}k.csv
```

Если нужно задать путь вручную:

```bash
python generate_data.py --num_samples 30000 --output storage/custom_dataset.csv
```

## Основные параметры генерации

### Размер и воспроизводимость

| Параметр | Значение |
|---|---|
| `--num_samples` | количество строк в датасете, должно быть кратно `1000` |
| `--seed` | seed генератора случайных чисел |
| `--output` | путь сохранения `.csv`; если не указан, строится автоматически |

### Физические параметры пластины

| Параметр | Значение по умолчанию | Смысл |
|---|---:|---|
| `--rho` | `7800.0` | плотность материала пластины |
| `--thickness` | `0.002` | толщина пластины |
| `--young_modulus` | `2.0e11` | модуль Юнга |
| `--poisson_ratio` | `0.3` | коэффициент Пуассона |

### Диапазоны случайной генерации

| Параметр | Смысл |
|---|---|
| `--omega_min`, `--omega_max` | диапазон циклической частоты |
| `--psi_min`, `--psi_max` | диапазон угла падения волны |
| `--dx_min`, `--dx_max` | диапазон горизонтального сдвига второй линии резонаторов |
| `--dy_min`, `--dy_max` | диапазон вертикального сдвига второй линии резонаторов |

### Параметры решетки и резонатора

| Параметр | Значение по умолчанию | Смысл |
|---|---:|---|
| `--period` | `0.05` | период массива резонаторов |
| `--n_terms` | `80` | число членов суммы в каждую сторону: `-n_terms, ..., n_terms` |
| `--omega0` | `2π · 100` | собственная частота резонатора |
| `--damping_ratio` | `0.01` | коэффициент демпфирования |
| `--mu_scale` | `1.0` | масштаб параметра взаимодействия резонатора с пластиной |

### Фильтрация физически допустимых строк

| Параметр | Смысл |
|---|---|
| `--max_R` | верхняя граница допустимого коэффициента отражения |
| `--max_T` | верхняя граница допустимого коэффициента прохождения |
| `--min_abs_denominator` | минимально допустимый модуль знаменателя в системе для амплитуд |
| `--max_abs_zeta` | максимально допустимый модуль амплитуд |
| `--max_attempt_multiplier` | максимальное число попыток генерации относительно размера выборки |

## Содержание датасета

В итоговый `.csv` сохраняются входные параметры, промежуточные физические величины и целевые значения:

```text
omega, psi, delta_x, delta_y, kappa,
Re_mu, Im_mu,
Re_S, Im_S, Re_S1, Im_S1, Re_S2, Im_S2,
Re_zeta1, Im_zeta1, Re_zeta2, Im_zeta2,
R, T, Re_R0, Im_R0,
energy_balance, abs_denominator, residual_1, residual_2
```

Для обучения PINN основными входами являются параметры частоты, геометрии, угла падения и физические суммы. Основной анализ качества строится по коэффициенту отражения `R`.

## Как запускать?

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Сгенерировать датасет нужного размера:

```bash
python generate_data.py --num_samples 30000
```

3. Открыть ноутбук:

```text
PINN/PhysicsInformedNN.ipynb
```

4. В `Config` ноутбука указать путь к датасету, например:

```python
 data_path = project_root / "storage" / "dataset_30k.csv"
```

5. Запустить ноутбук.

После запуска модель сохранится в `PINN/saved_models/`, а графики и таблицы — в `visualization/`.
